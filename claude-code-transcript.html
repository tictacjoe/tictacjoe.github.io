How this page was built — warts and all

The Claude Code demo page was designed and built in a single conversation. Below is the real transcript — including the mistakes, the dead deployment, the wrong assumptions, and the merge conflict. This is what working with an AI coding assistant actually looks like.

Total session time: about 20 minutes. Errors encountered: 4.

Designing the page

Joe

I'm showing someone Claude Code and could use some of my past work to make a presentation of the things that you can do.

Claude

I explored your project directories — homebot-twilio, jtda, plants, td, music-instruct — to understand what material was available. Then I asked: who is the audience?

Joe

My son is taking an intro AI course.

Claude

Asked: is this a live demo or a slide deck? How much time? What does he already know?

Joe

Live demo, 15–20 minutes. He built a CustomGPT in ChatGPT and has used Perplexity.

Claude

Proposed three approaches, recommended: open with the CustomGPT hook ("you built the GPT — I built the server it talks to"), then show memory, codebase explanation, and live code generation.

First problem: the deployment was dead

Claude

I checked whether the homebot-twilio Railway deployment was still live:

terminal
$ curl -s https://homebot-twilio-production.up.railway.app/healthz
response
{"status":"error","code":404,"message":"Application not found"}

The free-tier Railway deployment had expired. The code was all there locally — just no live server.

Claude

This didn't actually kill the demo. The codebase is still real and explainable. We just skip the "hit the live endpoint" moment and use it as a code walkthrough instead.

Building the page

Joe

Can you make a presentation he could see on his phone? I want to send him something now and follow up with a demo if he wants it.

Claude

Proposed GitHub Pages (shareable URL, already set up at tictacjoe.github.io). Joe approved. I checked the existing site structure and CSS before writing anything.

Joe

Don't include "how to reach me" — I'm right here. Add a comparison of ChatGPT vs Claude vs Gemini, the Claude model tiers, and a blurb about Anthropic's ethical AI commitment.

Second problem: wrong assumption about Jekyll

Claude
mistake

I saw a _layouts/ directory and a _config.yml in the repo and assumed this was a Jekyll site. I wrote the page with Jekyll front matter at the top:

--- layout: default title: What Claude Code Can Actually Do ---

I also added a meta viewport tag to _layouts/default.html, thinking that would fix mobile rendering.

Claude
discovery

Mid-build, I found a CLAUDE.md file inside the repo that said:

"Pure static HTML site — no Jekyll, no build step for the main site. GitHub Actions builds and deploys everything on push to main."

The _layouts/ directory was a leftover from a previous setup. Jekyll front matter would appear as raw text on the page.

Claude
fix

Rewrote the page as plain HTML using the template from the CLAUDE.md. Added the viewport tag directly in the page's own <head>. Reverted the unnecessary _layouts/default.html change.

Third problem: TLS error on push

Claude
error

First push attempt to GitHub failed with a network error:

error: unable to rewind rpc post data - try increasing http.postBuffer error: RPC failed; curl 56 GnuTLS recv error (-12): A TLS fatal alert has been received. fatal: the remote end hung up unexpectedly
Claude
fix

Retried the push. It went through on the second attempt — a transient WSL/network issue, not a code problem.

Fourth problem: master vs. main branch conflict

Claude
error

The local repo was on a master branch. GitHub had renamed the default branch to main at some point. When the push went through, it created a new master branch on the remote instead of updating main. GitHub Pages was serving from main, so the new page wasn't live yet.

Claude
fix

Fetched origin to get the remote main branch, then ran git merge master to bring the new page in. This triggered a merge conflict in a blog post file (blog/just-four-ducks-for-now.html) that had diverged between the two branches.

CONFLICT (content): Merge conflict in blog/just-four-ducks-for-now.html Automatic merge failed; fix conflicts and then commit the result.

Resolved by taking the remote main version of that file (it was unrelated to our changes), then committed the merge and pushed to main. Page went live.

End result

Claude

One conversation, four errors, about 20 minutes. The page is live at tictacjoe.github.io/claude-code.

This is what AI-assisted development actually looks like. The AI doesn't get everything right the first time — but it finds its mistakes, explains them, and fixes them without starting over.
