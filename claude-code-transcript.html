How this page was built

The Claude Code demo page was designed and built in a single conversation with Claude Code. Below is the actual back-and-forth — from the first message to a live URL.

This is a lightly edited transcript. The full session took about 20 minutes.

Joe

I'm showing someone Claude Code and could use some of my past work in Claude Code to make a presentation of the things that you can do.

Claude

I explored your project directories — homebot-twilio, jtda, plants, td, music-instruct — to understand what material was available. Then I asked: who is your audience?

Joe

My son is taking an intro AI course.

Claude

Is this a live demo or a slide deck he can look at on his own?

Joe

Show in real time in this interface.

Claude

How much time do you have — a few minutes or more like 20–30?

Joe

Maybe 15–20 minutes.

Claude

I proposed three approaches:

  • Full dev cycle: walk through homebot-twilio end to end
  • Capability sampler: quick hits across 5–6 distinct features
  • Bridge to his AI course: start with something he's studying, build from there

Recommendation: option 3 leading into option 2 — open with his coursework as a hook, then show real capabilities.

Joe

His course is basics — some AI ethics and social issues. He built a CustomGPT in ChatGPT. He also has some experience with Perplexity.

Claude

That's the hook: you built a CustomGPT — I built the server it talks to. I sketched a 15-minute arc: memory demo → homebot-twilio codebase tour → live code generation → ethics research tie-in → close.

Then I checked whether homebot-twilio was still live. It wasn't — Railway deployment returned a 404. The code was all there locally, though.

Joe

What else besides homebot-twilio might be interesting?

Claude

I suggested five options: the memory system, live code generation, web research with citations, GitHub MCP integration, and the td personal notes coding system. My pick for the strongest three-demo set: memory + codebase explanation + live build.

Joe

I like the 3 demo set.

Claude

Laid out the full 15-minute flow with timing for each demo and a closing line.

Joe

Can you make a presentation he could see on his phone as well?

Claude

Is this a companion during the demo, or something to share afterward?

Joe

I want to send him something now and then follow up with a demo if he wants it.

Claude

Three format options: GitHub Pages (shareable URL), a GitHub repo README, or a standalone HTML file. Recommended GitHub Pages — you already have tictacjoe.github.io set up.

Joe

Go with option 1.

Claude

Checked the existing site: Jekyll layout, clean CSS, already mobile-responsive. Proposed a seven-section page structure and got approval.

Joe

Don't include "how to reach me" — I'm right here. Also maybe a section about relative strengths and weaknesses of Gemini vs. Claude vs. ChatGPT, the different Claude flavors, and a blurb about Anthropic's commitment to ethical AI use.

Claude

Updated the design: dropped the contact section, added AI comparison, Claude model tiers, and Anthropic safety. Final structure confirmed.

Joe

Build it.

Claude

Discovered mid-build that the site is pure static HTML, not Jekyll — rewrote the page accordingly, resolved a merge conflict on the main branch, and pushed to GitHub Pages.

Total: one conversation, one URL.
