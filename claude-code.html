What Claude Code can actually do

You've built a CustomGPT. Claude Code is the tool that built the server it would talk to — and a lot more. Here's what it looks like in practice, using real projects.

1 — It remembers you

Unlike ChatGPT, Claude Code keeps a persistent memory file on your machine. Ask it about your projects, preferences, or past decisions and it answers from real notes — not just the current conversation.

real memory query — terminal output
$ claude "what are my active projects?" Your active projects: homebot-twilio FastAPI SMS relay on Railway (backend for a CustomGPT) jtda topological data analysis via quantum algorithms plants iHarvest hydroponics grow log, started 2025-12-15 music-instruct advanced harmony, songwriting, pedal board td personal notes and to-do tree, plain text files

2 — It understands real code

Ask it to explain an entire codebase and it reads every file, maps the architecture, and explains it in plain English. The project below is the server a CustomGPT calls to send a real SMS text message.

explaining a real project — terminal output
$ claude "explain this codebase like I'm a CS student" This is a FastAPI web server with one job: receive a request from a CustomGPT and send a real SMS via Twilio. Three endpoints: /healthz is the server running? /readyz are credentials configured? /send-sms send the actual text message Auth is an API key in the request header — only the CustomGPT that knows the secret can call it. Twilio credentials live in environment variables, never in the code itself. The whole thing deploys to Railway with one push.

3 — It builds things live

Name a topic and Claude writes working code around it on the spot. Here's a toy AI bias detector for job postings — written in about 10 seconds:

live code generation — terminal output
$ claude "write a toy bias detector for job descriptions" def check_bias(text): masculine = ["competitive", "dominant", "ninja", "rockstar"] feminine = ["collaborative", "supportive", "nurturing", "empathetic"] t = text.lower() m = sum(w in t for w in masculine) f = sum(w in t for w in feminine) if m > f: return f"Skews masculine ({m} signals)" if f > m: return f"Skews feminine ({f} signals)" return "Relatively neutral" # Try it: # check_bias("We need a competitive rockstar ninja") # → Skews masculine (3 signals)

ChatGPT vs. Claude vs. Gemini

Each tool has a genuine strength. Here's an honest read:

ChatGPT (OpenAI)
+ Largest ecosystem. Best plugin and CustomGPT builder tooling. Most people are already on it.
- Can be sycophantic — tends to agree with you rather than push back. Weaker on very long documents.
Claude (Anthropic)
+ Best for long documents, nuanced reasoning, and serious coding work. 200K token context window. Safety-first by design.
- Smaller ecosystem. No image generation. Fewer integrations than OpenAI.
Gemini (Google)
+ Deepest Google Workspace integration — works natively in Docs, Gmail, Sheets. Built-in search grounding.
- Less consistent reasoning. Often feels more like enhanced search than a thinking collaborator.

Claude models

Anthropic ships three tiers, each optimized for different tradeoffs between speed, cost, and capability.

Haikufastest · cheapest

Simple lookups, quick summaries, high-volume automation. Responds in under a second. Good for tasks where cost matters more than depth.

Sonnetbalanced · daily driver

Best price-to-performance ratio. What Claude Code runs by default. Strong enough for real development, writing, and research.

Opusmost capable

Complex reasoning, hard architectural decisions, deep research. Slower and more expensive — worth it when the problem is genuinely hard.

Anthropic and AI safety

Most AI companies treat safety as a compliance layer added after the fact. Anthropic was founded specifically because its researchers believed powerful AI needed to be built more carefully from the start.

Constitutional AI — Claude is trained against a written set of principles, not just human approval ratings. It's designed to reason about whether something is harmful, not just avoid flagged keywords.

Responsible Scaling Policy — Anthropic commits publicly to safety evaluations before shipping more powerful models. If a model crosses defined capability thresholds, development slows and audits happen before release.

Deliberate omissions — Claude has no image generation, in part to avoid contributing to synthetic media and deepfake harms at scale. These are conscious product decisions, not technical limitations.

Whether this fully solves the problem is genuinely debated — and Anthropic would say so too. But it's a meaningfully different posture than "ship fast, patch the ethics later."
